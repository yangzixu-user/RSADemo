Java中不同类型的密钥库-JKS

  皮克        2014-09-05 20:21:51       34,666        3    

JKS是Java密钥库，这是专为Java设计的专有密钥库类型。它可以用于存储用于SSL通信的私钥和证书，但是不能存储秘密密钥。JDK附带的keytool无法提取存储在JKS上的私钥。这种类型的密钥库通常具有jks的扩展名。

接下来，我们将展示如何使用纯Java代码来操作JKS密钥库。

创建JKS密钥库

创建JKS密钥库以创建空密钥库的最简单方法。我们首先可以获取KeyStore的实例，然后加载一个空的keystore。加载空密钥库后，我们只需要使用密钥库的名称和密码来调用KeyStore.store（）。

下面是一个简单的演示：

try{
    KeyStore keyStore = KeyStore.getInstance("JKS");
    keyStore.load(null,null);
     
    keyStore.store(new FileOutputStream("mytestkey.jks"), "password".toCharArray());
}catch(Exception ex){
    ex.printStackTrace();
}

执行完上述调用后，您将在当前工作目录中看到一个名为mytestkey.jks的密钥库。现在，密钥库为空，没有任何条目。

存储私钥

现在，让我们将一个私钥及其关联的证书链存储到密钥库中。请注意，如果没有关联的证书链，我们就无法使用JDK将私钥存储到密钥库中。使用某些其他库或本机库，您可能可以存储没有关联证书链的私钥。

try{
    KeyStore keyStore = KeyStore.getInstance("JKS");
    keyStore.load(new FileInputStream("mytestkey.jks"),"password".toCharArray());
     
    CertAndKeyGen gen = new CertAndKeyGen("RSA","SHA1WithRSA");
    gen.generate(1024);
     
    Key key=gen.getPrivateKey();
    X509Certificate cert=gen.getSelfCertificate(new X500Name("CN=ROOT"), (long)365*24*3600);
     
    X509Certificate[] chain = new X509Certificate[1];
    chain[0]=cert;
     
    keyStore.setKeyEntry("mykey", key, "password".toCharArray(), chain);
     
    keyStore.store(new FileOutputStream("mytestkey.jks"), "password".toCharArray());
}catch(Exception ex){
    ex.printStackTrace();
}

首先，我们将创建一个私钥和一个自签名证书，然后使用指定的别名，密钥，密钥的密码及其关联的证书链调用KeyStore.setKeyEntry（）。请记住，我们需要调用KeyStore.store（）将密钥存储到密钥库中。

别名是条目的标签，以便以后可以轻松找到它。

店铺证书

我们可以将证书存储在JKS密钥库中。要存储的证书应该是X509Certificate。可以将其存储在没有关联私钥的密钥库中。此过程类似于存储私钥。

try{
    KeyStore keyStore = KeyStore.getInstance("JKS");
    keyStore.load(new FileInputStream("mytestkey.jks"),"password".toCharArray());
     
    CertAndKeyGen gen = new CertAndKeyGen("RSA","SHA1WithRSA");
    gen.generate(1024);
     
    X509Certificate cert=gen.getSelfCertificate(new X500Name("CN=SINGLE_CERTIFICATE"), (long)365*24*3600);
     
    keyStore.setCertificateEntry("single_cert", cert);
     
    keyStore.store(new FileOutputStream("mytestkey.jks"), "password".toCharArray());
}catch(Exception ex){
    ex.printStackTrace();
}

加载私钥

存储密钥后，我们还可以将条目加载到密钥库中。在这里，我们说要加载私钥，实际上这里不是这种情况，正如我们前面所描述的，不能使用Java从JKS中提取私钥。在这里，我们实际上提取了私钥的证书链。

    keyStore.load(new FileInputStream("mytestkey.jks"),"password".toCharArray());
     
    Key key = keyStore.getKey("mykey", "password".toCharArray());
//          System.out.println("Private key : "+key.toString());   //You will get a NullPointerException if you uncomment this line
     
    java.security.cert.Certificate[] chain =  keyStore.getCertificateChain("mykey");
    for(java.security.cert.Certificate cert:chain){
        System.out.println(cert.toString());
    }
}catch(Exception ex){
    ex.printStackTrace();
}

导入密钥和证书

这个过程实际上非常简单，我们首先需要在要导入证书的地方加载密钥库。然后，我们还需要加载另一个密钥库，我们需要将证书导入到该密钥库。接下来，我们需要从源密钥库中获取证书并将其放入目标密钥库中。

由于我们无法从JKS提取私钥，因此我们只能将证书导入JKS。但是，我们可以从其他类型的密钥库（PKCS12）中提取私钥，然后将其存储在JKS密钥库中。

最后一条信息。Oracle提供了两种JKS密钥库版本：区分大小写和不区分大小写。调用KeyStore.getInstance（“ JKS”）时，将创建不区分大小写的JKS实例版本，当调用KeyStore.getInstance（“ CaseExactJKS”）时，将创建JKS实例的区分大小写版本。通常建议不区分大小写，因为用户应区分不同别名的条目，而不是别名的大小写。有关区分大小写的更多信息，请参阅此文章。
